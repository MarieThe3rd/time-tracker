# Task Manager Stories and Complexity

## Story TM-001 Domain model and persistence
**Complexity:** 8

Create task models, enums, validation rules, persistence structures, and migration strategy.

### Includes
- task entity and supporting models
- task type enum
- task status enum
- date validation rules
- deliverable target modeling
- migration or storage updates

## Story TM-002 Task CRUD screens
**Complexity:** 8

Create list and edit/create experiences for tasks in the Blazor UI.

### Includes
- create form
- edit form
- list view
- basic filtering
- validation messages

## Story TM-003 Task workflow rules
**Complexity:** 5

Implement status transitions, completion date behavior, and active/archive defaults.

## Story TM-004 Reporting queries
**Complexity:** 5

Add query services for active tasks, due soon, completed by range, ideas, and squirrels.

## Story TM-005 Obsidian export
**Complexity:** 8

Build markdown export service and report templates suitable for the vault.

## Story TM-006 Automated tests
**Complexity:** 8

Add domain, service, component, and regression tests for the epic.

## Story TM-007 Documentation and peer review
**Complexity:** 3

Update docs, add delivery notes, and record peer review findings.
