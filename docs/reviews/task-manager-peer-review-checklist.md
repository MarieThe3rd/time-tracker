# Task Manager Peer Review Checklist

## Requirement alignment

- task types implemented
- task statuses implemented
- required and optional fields supported
- Obsidian export included
- reporting included

## Code quality

- boundaries are clean
- naming is clear
- rules are centralized
- no obvious duplication or hidden coupling

## Tests

- domain rules covered
- UI or component behavior covered where needed
- export formatting covered
- regression tests added for defects

## Docs

- requirements updated
- architecture notes updated
- testing strategy updated
- delivery notes added if needed

## Risks to check

- invalid or conflicting dates
- archived visibility bugs
- incomplete export content
- persistence migration issues
- performance on large task lists
